Configuring Services

 
Some services have a set of parameters that you can edit, for example, the number of seconds for timeouts.

Setting the timeout

To configure services

1 On the Manage Servers Page, select the server and click Configure.

3 Modify any of the property information on the page (see the table below for a description of the properties).

4 Click Save.

The server immediately starts processing according to the changes you made.
Site Service  
Session timeout (seconds) Time duration in seconds for when an idle connection is dropped
Session timer interval (seconds) Time interval in seconds for when the server checks for idle connections
Resource Service  
Data file trash folder Folder where the repository trash files are stored
Library data file folder Path where the Library resource data files are stored
Library repository folder Path where the Library repository is stored
Session data file folder Path where the Session resource data files are stored
Session repository folder Path where the Session repository is stored
Site repository folder Path where the Site repository is stored
Resource schema folder Path where the resource schema files are stored
Repository checkpoints timer interval (seconds) Time in seconds for when the server performs checkpoints for all the repositories (value <= 2147483647)
Packages folder Path where the resource packages can be found
Feature Service  
Maximum connections pooled per provider (default) Default number of connections to cache
Maximum connections pooled per provider (custom) Custom number of connections to cache (value <= 1024)
Data connection timeout (seconds) Time duration in seconds for when an idle connection is dropped (value <= 86400)
Data connection timer interval (seconds) Time interval in seconds for when the server checks for idle connections (value <= 2147483647)
Maximum features cached Max number of features to fetch (value <= 2147483647)
Tile Service  
Tile cache path
 Root path of the image tile cache
