MapGuide Open Source - Features

MapGuide Open Source is a web-based platform that enables users to quickly develop and deploy web mapping applications and geo-spatial web services. It is comprised of four major components, the MapGuide Server, MapGuide Web Server Extensions, MapGuide Viewer, and the MapGuide Site Administrator.

MapGuide Server Features

The functionality of the MapGuide Server is broken down into a collection of services each of which adheres to the principles of service-oriented archtecture. The core features of the server include:

  • Runs as a Daemon on Linux or as a Service on Microsoft Windows.
  • Servers can be grouped together into a "Site" to increase scalability.
  • Provides connection oriented, role based security.
  • Provides configurable logging of client access, administrative access, authentication attempts, errors, session lifetimes, and request tracing.

MapGuide Server

Broad Platform Support
Runs as a service under Microsoft® Windows Server® using IIS or Apache and as a Daemon under Linuxsing Apache.

Server-Side Application Development and Delivery
Write applications that support both raster and vector viewing with the same code.

Multilanguage Support
Use .NET, JavaScript, or PHP through the Web Server Extensions, with 100 percent consistency between API (application programming interface) calls.

Open GeoSpatial Agent
MapGuide Open Source exposes data and layers via the Open GeoSpatial Web Map Services (WMS) and Web Feature Services (WFS), thus providing map layers to other applications.

MapAgent Service
Easy set up and configuration for both secure and nonsecure applications. It accepts requests from clients over standard HTTP or HTTPS protocols and runs as a CGI/FastCGI to simplify web server configuration.

Simultaneous Connections
Supports simultaneous connections (parallel or simultaneous queries) to multiple database servers, residing locally or on networked Unix or Windows systems.

Built-in, Access-Based Security Model
Flexible control over information distribution, helping to ensure delivery of your data or application to authorized users only.

Scalability
Takes full advantage of multiprocessor architectures, allowing a site to grow to support large numbers of concurrent users. Servers can be added to provide specific services, such as raster map rendering.

Web-Based Administrator
Remotely administer servers from any browser. Add and remove servers, configure servers and services, start and stop servers, configure log generation, view log files, and define users and groups.

MapGuide Viewer

Flexible Viewing Options
Two map viewing options provide similar tools:

  • Autodesk® DWF™ Viewer—a downloadable ActiveX control that displays vector-based maps on Windows systems running the Internet Explorer or Firefox™ browser
  • AJAX Viewer—delivers raster-based maps to almost any browser

Portability
DWF Viewer supports “disconnected” mode, providing off-line vector-based map viewing capabilities that maintain data attributes, allowing map data to be taken into the field for viewing or markup.

Multiple Object Selection
Select objects by list, radius, polygon, buffer, and intersection.

Property Browser
The viewer displays the properties of a selected object as defined by the map author, providing immediate access to essential attribute data.

Customization
Customize the DWF Viewer for use within websites and third-party applications using the DWF Viewer API. Navigate to a specific page or view, control layer visibility, show and hide the toolbar and context-sensitive menus, and customize viewer behavior.

Quality Printing/Plotting
DWF support lets users produce high-quality presentation materials. There’s support for multiple paper sizes, print preview, multiple scale options and control of elements to print on a page, including plot scale, title, legend, map scale, north arrow, URL, current date and time—even a logo mark.

Autodesk MapGuide Studio and Web Server Extensions

Unified Authoring Environment
Studio provides a developer-friendly authoring environment based on popular web development tools provides flexibility, ease of use, fast prototyping, and rapid application development.  Studio eliminates the need to use different tools for loading data, connecting to databases, and publishing data to the web. Autodesk MapGuide Studio handles all aspects of collecting and preparing geospatial data, including:

  • Use of Site Explorer—MDI tabbed window
  • Ability to build stylized and themed layers
  • Ability to compile the layers into a map
  • Display attributes defined by scale
  • Automatic labeling by scale
  • Web layout preview without publishing
  • Customizable popup menu
  • Write application inside frame (native application development)
  • And much more

Efficient Data Access
Uses a single API—Feature Data Objects (FDO)—for accessing your data in ODBC, ArcView™ Shape file, SDF+ file, Oracle® Spatial DB, ArcSDE™ database, and other formats.

Raster Data Management
Supports most raster formats, including BMP, CAL, ECW, JPG, PNG, SID, TGA, and TIF.

WYSIWYG Display of Authored Maps
Easy-to-use interface provides preview of web layout as well as immediate feedback when authoring.

Dual Viewer Applications
Create a single application that works with both the DWF Viewer and the AJAX Viewer, for simplified publishing without extra steps.

Standards Support
Open Geospatial FDO Providers allow use of WMS and WFS as map layers for your application.
                       
.Net API
Programmatically load and organize data from any compatible language to quickly create programs to do repetitive work.
